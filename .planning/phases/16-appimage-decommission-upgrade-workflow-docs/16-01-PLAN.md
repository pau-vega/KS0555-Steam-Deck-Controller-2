<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → GitHub Releases API | CI uploads release assets using GITHUB_TOKEN |
| CI runner → crates.io / npm registry | Build-time dependency fetching |
| CI runner → Flathub OSTree cache | flatpak-builder fetches runtime from Flathub |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Tampering | build.yml release upload step | mitigate | `softprops/action-gh-release@v2` uses GITHUB_TOKEN scoped to the repo; `contents: write` is step-level only, not job-level |
| T-16-02 | Spoofing | `cargo install tauri-cli` | accept | Installing from crates.io via HTTPS — standard Tauri practice. Risk accepted as no alternative (Tauri doesn't ship pre-built binaries for CI). |
| T-16-03 | Elevation | `sudo apt-get install` in CI | accept | CI runs on ephemeral GitHub-hosted runners — sudo is expected for system deps. Runner is destroyed after job completion. |
| T-16-04 | Denial | OSTree cache in flatpak-builder | mitigate | OSTree cache uses `cache: true` in the action; GitHub Actions cache has 10GB limit — cache eviction is benign (rebuilds take longer but don't fail). Cache key includes runtime version to prevent stale cache poisoning. |
</threat_model>
