<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner → GitHub Release API | `GITHUB_TOKEN` authenticates release asset upload; scoped to `contents: write` in `build-flatpak-x64` only |
| `build-x64` artifact → `build-flatpak-x64` download | Workflow artifact passing; GitHub manages integrity of artifact storage/retrieval |
| `build-flatpak-x64` → `flatpak-builder` container | Official `ghcr.io/flathub-infra/flatpak-github-actions:freedesktop-24.08` container; the `.deb` and repo sources are mounted read-only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | `.deb` artifact between jobs | accept | GitHub-managed artifact storage; integrity is GitHub's responsibility within the same workflow run |
| T-15-02 | Elevation | `contents: write` permission | mitigate | D-04 — job-level `contents: write` only on `build-flatpak-x64`; `build-x64` and top-level are `contents: read` |
| T-15-03 | Information Disclosure | OSTree cache | accept | Cache is GitHub-managed; no secrets or PII stored in Flatpak build cache |
