<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend ↔ Tauri Backend | Frontend calls Tauri commands via invoke(), receives events via listen() |
| Tauri Backend ↔ System | Rust code accesses Bluetooth (btleplug) and gamepad (gilrs) hardware |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-6-01 | Spoofing | Tauri commands (ble_connect, ble_send) | mitigate | Tauri v2 permissions model restricts which commands can be called from frontend; configure permissions/default.toml |
| T-6-02 | Tampering | Frontend → Tauri IPC | mitigate | Tauri's invoke() uses typed commands; enable CSP in tauri.conf.json to restrict script execution |
| T-6-03 | Information Disclosure | Dev tools in production | mitigate | Disable devtools feature in Cargo.toml for production builds (currently enabled for debugging per D-10) |
| T-6-04 | Denial of Service | Tauri dev server port 5173 | accept | Development only; production uses packaged AppImage. Port conflict unlikely in single-user Steam Deck scenario |
| T-6-05 | Elevation of Privilege | Tauri backend system access | mitigate | Tauri runs with user-level permissions; btleplug and gilrs require bluetooth/gamepad access which is appropriate for this app's purpose |

</threat_model>
