# Quick Task 260511-001: Fix D-pad not working in gaming mode

**Created:** 2026-05-11
**Mode:** quick

## Goal

The gamepad module only reads `Axis::LeftStickX`/`LeftStickY`. The Steam Deck D-pad
reports via `Axis::DPadX`/`DPadY` (analog axes) or `Button::DPadUp`/`Down`/`Left`/`Right`
(hat switch). Neither is handled → D-pad does nothing in the app.

## Tasks

### Task 1: Extend gamepad monitor to handle D-pad input

**Files:** `apps/frontend/src-tauri/src/gamepad/mod.rs`

**Action:**
1. Import `Button` from gilrs
2. Extract `compute_direction` helper that checks D-pad first (axes + buttons),
   falls back to LeftStick
3. Add `Axis::DPadX`/`Axis::DPadY` to the `AxisChanged` filter
4. Add `ButtonChanged`/`ButtonPressed`/`ButtonReleased` handler for D-pad buttons
5. When D-pad is active (any axis non-zero or any D-pad button pressed), use
   D-pad direction — otherwise use stick

**Verify:** `cargo test -p tauri-app` passes all gamepad tests

**Done:** All unit tests pass, new tests cover D-pad inputs
