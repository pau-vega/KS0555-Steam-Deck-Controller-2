# Quick Task 260511-001: Fix D-pad not working in gaming mode

**Completed:** 2026-05-11
**Status:** complete

## Summary

The gamepad monitor only read `Axis::LeftStickX`/`LeftStickY`. The Steam Deck
D-pad reports via `Axis::DPadX`/`DPadY` (analog axes) or `Button::DPadUp/Down/
Left/Right` (hat switch) — neither was handled, so D-pad did nothing.

### Changes

**`apps/frontend/src-tauri/src/gamepad/mod.rs`:**
1. Added `Button` to gilrs imports
2. Extracted `compute_direction()` helper that checks D-pad first (axes + buttons),
   falls back to LeftStick
3. Extended AxisChanged handler to also match `Axis::DPadX`/`DPadY`
4. Added handler for `ButtonChanged`/`ButtonPressed`/`ButtonReleased` on D-pad buttons
5. D-pad takes priority when actively used; stick works normally otherwise

### How it works

- When D-pad has any input (axis non-zero or any DPAD button pressed), use D-pad
- Otherwise, fall through to LeftStick axes (existing behavior)
- Direction change guard prevents event spam
- Deadzone (0.15) applies to both D-pad axes and stick
