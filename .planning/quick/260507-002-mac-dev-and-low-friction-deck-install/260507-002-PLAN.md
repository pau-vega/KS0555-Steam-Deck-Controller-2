# Quick Task Plan: Mac Dev + Low-Friction Steam Deck Install

## Context

Current state: Tauri v2 app with Rust BLE (btleplug) + gamepad (gilrs). AppImage builds via CI (`.github/workflows/build.yml`) on tag/dispatch. README still references the legacy pre-Tauri Node backend (bluetoothctl + rfcomm + Chrome Gamepad API + WebSocket). Bundle targets are Linux-only (`["appimage"]`).

User goal: develop and test the app on Apple Silicon Mac without booting the Steam Deck for every iteration; on Steam Deck, the install should be near zero-friction.

## Task 1: Add macOS bundle support to Tauri config
- **files**: `apps/frontend/src-tauri/tauri.conf.json`, `apps/frontend/src-tauri/Info.plist` (new)
- **action**:
  - Change `bundle.targets` from `["appimage"]` to `"all"` so Tauri picks the right bundle per host (DMG/.app on macOS, AppImage/deb on Linux).
  - Add `bundle.macOS.minimumSystemVersion: "11.0"` (Big Sur — required for CoreBluetooth and Apple Silicon support).
  - Create `src-tauri/Info.plist` declaring `NSBluetoothAlwaysUsageDescription`. btleplug requires this on macOS ≥ Big Sur or BLE silently fails. Tauri auto-merges this Info.plist into the bundle.
- **verify**: `tauri.conf.json` validates against schema; `Info.plist` parses with `plutil -lint`.
- **done**: false

## Task 2: Smoke-test Mac build chain
- **files**: none (verification only)
- **action**: From `apps/frontend/src-tauri/`, run `cargo check --target aarch64-apple-darwin` to confirm Rust deps (btleplug, gilrs, tokio) compile on Mac. Skip full `tauri build` (~10 min cold) — `cargo check` proves the toolchain is wired.
- **verify**: `cargo check` exits 0.
- **done**: false

## Task 3: Attach AppImage to GitHub Releases on tag push
- **files**: `.github/workflows/build.yml`
- **action**: Replace `actions/upload-artifact` (CI-only) with `softprops/action-gh-release` so tagged builds publish AppImages to a real GitHub Release. Keep `actions/upload-artifact` for manual `workflow_dispatch` runs (no tag → no release). Net effect: Steam Deck users get a stable download URL like `https://github.com/<repo>/releases/latest/download/robot-controller-x86_64.AppImage`.
- **verify**: workflow YAML validates; conditional logic uses `github.ref_type == 'tag'` correctly.
- **done**: false

## Task 4: Create `install-on-steamdeck.sh`
- **files**: `install-on-steamdeck.sh` (new, repo root)
- **action**: One-shot installer that:
  1. Detects arch (x86_64 vs aarch64).
  2. Curls the matching AppImage from `https://github.com/<repo>/releases/latest/download/...`.
  3. Drops it in `~/Applications/RobotController.AppImage`.
  4. `chmod +x`.
  5. Writes a `.desktop` file in `~/.local/share/applications/` so it shows up in KDE's app menu (and in `Add Non-Steam Game` picker).
  6. Prints final instructions: open Steam → Library → "+" → "Add a Non-Steam Game" → pick `Robot Controller`.

  Designed to be run via `curl -fsSL https://raw.githubusercontent.com/<repo>/main/install-on-steamdeck.sh | bash` from Konsole on the Deck. The repo placeholder is filled with `pvelasco/KS0555-Steam-Deck-Controller-2` — adjust if the repo gets renamed.
- **verify**: `bash -n install-on-steamdeck.sh` (syntax check); script uses `set -euo pipefail`; falls back gracefully if curl/release missing.
- **done**: false

## Task 5: Rewrite README install + dev sections
- **files**: `README.md`
- **action**:
  - Drop the legacy pre-Tauri sections: `bluetoothctl` pairing, `rfcomm bind`, Chrome Gamepad / Flatpak override, `pnpm dev:frontend`/`dev:backend` split, the `ws://localhost:8080` architecture diagram, and the `WS_PORT/SERIAL_PORT` env table — none apply since BLE moved into Rust.
  - Add three sections:
    1. **Install on Steam Deck** — the one-line `curl … | bash` plus the manual fallback (download AppImage from Releases → chmod +x → Add Non-Steam Game).
    2. **Develop on Mac** — `pnpm install`, `pnpm --filter @ks0555/frontend tauri:dev`. Note that the OS will prompt for Bluetooth access on first BLE scan (driven by the `NSBluetoothAlwaysUsageDescription` we just added).
    3. **Architecture (current)** — `Frontend (React/Vite) ←→ Tauri IPC ←→ Rust (btleplug + gilrs) ←→ BT24 ←→ Arduino`. Single binary; no separate backend process.
  - Keep gamepad-mapping table and command table.
- **verify**: README does not mention `rfcomm`, `bluetoothctl`, `flatpak override`, `dev:backend`, or `WS_PORT`.
- **done**: false

## Out of scope
- Not removing the legacy `apps/backend/` package (orphan but harmless; separate decision).
- Not signing / notarizing the macOS bundle — local dev only; users will see the unsigned-app warning on first run, which is acceptable for a personal tool.
- Not adding macOS to the CI matrix — Mac builds happen locally on the user's machine.
- Not rewriting `build-steamdeck.sh` (kept for offline / source-build fallback).
