# Quick Task Summary: Per-Device Run Instructions

## What changed

- `docs/RUNNING.md` (new) ‚Äî canonical per-device run guide. Three sections:
  - **Steam Deck end-user**: curl one-liner install, manual install fallback, Add-Non-Steam-Game flow, Gaming Mode launch, on-device source build via `build-steamdeck.sh`, update flow.
  - **macOS developer**: prereqs (Big Sur+, Xcode CLT, Node, pnpm, Rust), `pnpm install && pnpm dev`, walk-through of the `NSBluetoothAlwaysUsageDescription` prompt and how to recover from a denied permission, local `tauri:build` for `.app`/`.dmg`, Gatekeeper unsigned-app bypass.
  - **Linux developer**: pacman + apt prereq commands, dev mode, `input` group requirement for evdev/gilrs access, local AppImage build.
  - **Troubleshooting matrix** covering 7 common failure modes (BT24 silent, Mac BT denied, Gaming Mode crash, deadzone showing `S`, BLE write succeeds but robot idle, macOS linker errors after upgrade, Gamescope evdev visibility).
  - **File map** pointing to the canonical source for each concern.
- `docs/STEAM_DECK.md` ‚Äî corrected the long-standing arch error. Steam Deck APU is AMD **x86_64**, not aarch64. Build commands and CI snippet now use the x86_64 toolchain. Noted that the existing `ubuntu-22.04-arm` job in `build.yml` targets ARM SBCs, not the Deck.
- `README.md` ‚Äî added a one-line callout under the project description linking to `docs/RUNNING.md`. Existing short install/dev sections preserved.
