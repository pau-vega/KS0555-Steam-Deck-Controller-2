  <action>Fix the TS6059 error in apps/backend/tsconfig.json.

Current issue: `"include": ["src", "vitest.config.ts"]` with `"rootDir": "src"` causes TypeScript error because vitest.config.ts is not under rootDir.

Fix: Remove "vitest.config.ts" from the include array since test config files don't need to be type-checked as part of the main compilation. The vitest.config.ts file is a configuration file that will be processed separately by Vitest.

Updated tsconfig.json should be:
```json
{
  "extends": "@ks0555/tsconfig/tsconfig.node.json",
  "compilerOptions": {
    "outDir": "dist",
    "rootDir": "src"
  },
  "include": ["src"],
  "exclude": ["node_modules", "dist"]
}
```

This removes vitest.config.ts from include, eliminating the TS6059 error while keeping all source files under src/ in the compilation.
  </action>
  <verify>
    <automated>pnpm typecheck 2>&1 | grep -q "TS6059" && echo "FAIL: TS6059 still present" || echo "PASS: TS6059 fixed"</automated>
  </verify>
